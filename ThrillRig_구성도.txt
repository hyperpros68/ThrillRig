가. 각 시스템간 신호 종류 (Master: Protocol.json)
  1. Call(신호) [F13] : 원격 점수 투입. 게임 점수 개수를 확인하여 플레이 점수 반영.
  2. Control(제어) [F14] : 배출 차단 및 시간 제어. 분 단위 설정값 전달.
  3. Emergency(응급) [EMG] : PC 리부팅(PC_RESET), 전원종료(PC_OFF), 게임종료(GAME_KILL).
  4. Statics(정산) [STAT] : 이용금액(Credit) 및 뱅크 사용 정산 데이터 보고.
  5. Billing(과금) [BILL] : 과금 및 결제 트랜잭션 처리.
  6. ACK(확인) [ACK] : 메시지 수신 확인 응답 (등기 방식 통신용).

나. 신뢰성 통신 구조 (Reliable Messaging Architecture)
  - 방식: Application Level Acknowledge (등기 우편 방식)
  - Transaction ID (TranID) 관리:
    * 송신측(Adapter)에서 일련번호(Sequence) + 데이터 해시를 조합한 고유 ID 생성.
    * 송신측 레지스트리에 마지막 Sequence 번호를 저장하여 재시작 시에도 중복 방지.
  - 재전송 로직:
    * 메시지 발송 후 3초 이내에 ACK 응답이 없으면 최대 3회 자동 재전송.
  - 중복 수신 방지 (Idempotency):
    * 수신측(Brain, Web)에서 최근 처리한 TranID를 캐싱하여 중복 메시지는 ACK만 보내고 로직 무시.
